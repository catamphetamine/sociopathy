Сначала запрос попадает на NginX.
NginX отдаёт файл "основа.html".
Он не отдаётся сам по себе, т.к. он закеширован, и достаётся у пользователя из кеша обозревателя.
Далее, в этом файле идёт ajax запрос на "инициализацию" (/initialize) на Node.js, и получаются все нужные данные, включая версию приложения.
Версия приложения хранится в файле version.coffee.
К любым файлам (javascript, css), загружаемым после этого, в конце добавляется версия, которая позволяет обновлять закешированную у пользователя "статику" при обновлении версии сайта.
Загружается скрипт основа.js.
Он сначала подгружает основные скрипты типа jQuery.
Затем подгружает все остальные скрипты (включая скрипты плагинов) (список подгружаемых скриптов прописан в файле основа.js).
Затем подгружаются и компилируются стили (список подгружаемых стилей прописан в файле основа.js).
Далее уже смотрится, какой Url прописан в адресной строке, и в зависимости от этого загружается нужная страница (navigation.js).

Бекенд написан на Node.js.
База данных - MongoDB.
Для прочих данных используется Redis.

таблица people_sessions:

 * последние_сообщения — последние сообщения из тех общений, на которые подписан пользователь
 * последние_прочитанные_сообщения — последние прочитанные сообщения из всех общений, которые когда-либо читал пользователь
 
при удалении общения нужно удалять из этих двух полей тоже
