Для того, чтобы написать плагин, нужно:

1) создать папку с названием плагина в папке plugins
2) заполнить ему файл info.json (кодировка: "utf-8 без BOM")
3) создать папку code с серверным кодом плагина
4) в файле info.json указать название главного файла с кодом (свойство "code");
     тип файла: js или coffee
5) всю "статику" положить в static resources/plugins, в папку с названием плагина
6) если нужен значок в навершии — прописываем путь к нему в свойстве "icon.picture" (формат — png);
     ссылку, по которой будет вести значок плагина, прописываем в свойстве "icon.to";
	 если значок должен быть доступен только для прописанных пользователей — ставим ему свойство "icon.private" в true;
	 если значок должен меняться в зависимости от каких-то условий (например, при получении новых "непрочитанных" данных), то это тоже можно сделать
7) если у плагина есть страницы, то объявляем их в массиве "pages",
     и кладём их шаблоны, скрипты и стили в папку pages

=================================================

в той папке плагина, которая лежит в static resources:

* перевод плагина (если он есть) должен лежать в папке translations
* иконка (если она есть) должна лежать в папке icon
* все картинки (если они есть) должны лежать в папке pictures

* все прочие клиентские яваскрипты должны лежать в папке "client scripts"
   (система найдёт и вставит их на страницу автоматически)
* все прочие шаблоны html должны лежать в папке templates
   (система найдёт и скомпилирует их под этими именами автоматически)
